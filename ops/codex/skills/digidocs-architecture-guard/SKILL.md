# Skill: digidocs-architecture-guard

## 目的

在 DigiDocs Mgt 中执行“文档优先、契约优先、边界优先”的架构守卫，避免实现静默偏离设计。

## 触发条件

- 需求涉及表结构、状态机、角色、API、跨系统集成
- 需求描述中出现 `设计文档`、`契约`、`schema`、`API`、`状态`
- 开发者对当前实现是否偏离设计存在不确定性

## 必读信息源

按顺序读取：

1. `docs/项目定义与技术架构.md`
2. `docs/数据库设计.md`
3. `docs/API设计.md`
4. `AGENTS.md`
5. `README.md`
6. `TASKS.md`

## 操作要求

- 先确认改动影响哪份设计文档，再开始代码改动。
- 如果实现将偏离设计，先更新设计文档，再更新代码。
- 禁止私自新增角色、状态、跨系统调用方式。
- 涉及群晖或 OpenClaw 时，必须确认仍经由适配层或客户端模块统一接入。

## 输出格式

开始实现前，先明确：

1. 影响的设计文档
2. 影响的数据契约或 API 契约
3. 需要更新的状态账本
4. 基础验证方式

## 完成检查

- 设计文档与实现一致
- `README.md` 和 `TASKS.md` 已同步
- 响应或总结中说明验证方式
