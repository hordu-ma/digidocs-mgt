# DigiDocs 文档资产管理平台 · 用户使用说明

本文面向**团队负责人**和**团队成员**，说明如何使用平台的各项功能。

---

## 1. 系统简介

DigiDocs 是面向课题组的文档资产管理与智能助理平台。核心目标是：

- **文档可留**：所有文档归属团队空间，不依赖个人电脑保存；
- **过程可查**：文档的版本、责任人变更、流转过程全程可追溯；
- **交接可管**：毕业交接形成标准流程，保障资料不断档；
- **知识可用**：借助 AI 助手对文档进行摘要、问答和建议输出。

平台分为以下模块：**总览**、**文档**、**数据**、**交接**、**助手**，管理员另有**管理**模块。

---

## 2. 登录与个人信息

### 登录

在浏览器中打开平台地址，输入用户名和密码后点击**登录**。账号由管理员统一创建。

### 个人信息维护

登录后，点击右上角**用户名 → 个人信息**，可修改：

| 字段 | 说明 |
|------|------|
| 显示姓名 | 在平台各页面显示的名称（必填） |
| 邮箱 | 联系方式，选填 |
| 手机 | 联系方式，选填 |
| 微信号 | 联系方式，选填 |

修改后点击**保存**即可生效。

---

## 3. 总览（Dashboard）

**适合对象**：负责人日常首屏查看。

进入平台后默认显示**总览**页，包含以下信息：

| 卡片 | 含义 |
|------|------|
| 文档总量 | 纳入平台管理的文档总数 |
| 处理中 | 当前仍在责任人处理中的文档数 |
| 待交接 | 尚未完成的毕业交接单数 |
| 风险文档 | 长期未更新或交接异常的文档数，需重点关注 |

页面下方还展示：

- **近期流转记录**：谁对哪份文档做了什么操作，时间一目了然；
- **风险文档列表**：点击文档名可直接跳转到文档详情页处理。

## 4. 文档（Documents）

### 4.1 文档列表

侧边栏点击**文档**进入文档列表，按课题（项目）分组展示。

**常用操作：**

- **搜索**：在页面顶部输入关键词，可跨组过滤；
- **快速筛选**：点击"全部 / 处理中 / 待交接 / 定稿 / 草稿"快速切换；
- **显示已归档**：开启开关后显示已归档文档（灰色半透明），关闭后仅显示活跃文档；
- **新建文档**：点击右上角**新建文档**按钮，填写标题、选择课题、目录，并上传初始文件（支持 docx/xlsx/pptx/pdf/txt/md 等格式）；
- **进入详情**：点击文档标题进入文档详情页。

### 4.2 文档详情

文档详情页分为三个区域：

#### 基本信息区

显示文档标题、当前状态、当前版本、当前责任人。

- **档案编辑**：修改文档标题、描述或所属目录；
- **删除文档**：将文档标为已删除，可在列表通过"显示已归档"恢复。

#### 流转操作区（根据当前状态动态显示）

| 当前状态 | 可执行操作 |
|----------|------------|
| 草稿 | 开始处理（进入"处理中"） |
| 处理中 | 转交给其他成员 / 定稿 |
| 待交接（转交中） | 接受转交 |
| 定稿 | 归档 |
| 已归档 | 取消归档（恢复为定稿） |

**转交**时需在下拉列表中选择接收人，对方接受后责任归属变更并记录在流转历史中。

#### 版本与 AI 区

- **上传新版本**：上传修订后的文件，填写版本说明（Commit Message）；
- **下载当前版本**：下载最新版文件；
- **版本历史**：展示所有历史版本，每行可单独下载；
- **AI 摘要**：点击**生成摘要**，AI 将自动提取文档正文并生成摘要，结果回写到版本记录；
- **AI 建议**：AI 分析后会生成操作建议（如"建议补充数据来源"），可对每条建议选择**确认**或**忽略**；
- **流转记录**：展示本文档的完整流转历史。

## 5. 数据（Data）

用于管理**非文档类**的数据文件（如数据集、模型文件、代码压缩包等），支持任意文件格式。

### 5.1 左侧导航

- 上方为**团队空间**和**课题**下拉选择器；
- 下方为当前课题下的**文件夹树**，点击可进入对应文件夹；
- 点击文件夹右侧的删除图标可删除空文件夹；
- 点击**新建文件夹**可在当前课题下创建子目录（支持多级）。

### 5.2 文件列表

右侧展示选中课题/文件夹下的数据文件，支持：

- **关键词搜索**；
- **文件类型筛选**（图片、视频、压缩包、数据等）；
- **上传文件**：点击**上传文件**按钮，通过点击或拖拽选择文件，选择所属课题和目录后提交（支持大文件，带进度显示）；
- **下载**：点击每行右侧的下载图标下载文件；
- **删除**：点击删除图标并确认后删除。

---

## 6. 交接（Handovers）

用于管理**毕业交接**流程，确保成员离组前完整移交文档和数据资产。

### 6.1 交接列表

显示所有相关的交接单，支持按状态筛选（全部 / 进行中 / 已完成 / 已取消）和排序。

**交接单状态说明：**

| 状态 | 含义 |
|------|------|
| 待确认 | 已创建，等待接收方确认 |
| 进行中 | 接收方已确认，资产交接处理中 |
| 已完成 | 交接双方确认完成 |
| 已取消 | 交接已取消 |

### 6.2 创建交接单

点击右上角**创建交接**，填写：

- **移交人**：发起交接的成员；
- **接收人**：接收资产的成员；
- **所属课题**：选填；
- **备注**：选填说明。

创建后进入编辑清单阶段。

### 6.3 编辑交接清单

点击交接卡片的**编辑**或**查看详情**，弹出详情窗口，分**文档**和**数据资产**两个 Tab：

- 勾选需纳入交接的文档和数据文件；
- 可为每一项填写交接备注；
- 点击**保存清单**保存修改。

### 6.4 交接流程推进

| 操作 | 触发条件 |
|------|----------|
| **确认交接**（接收方操作） | 清单已填写完毕，接收方确认开始处理 |
| **完成交接** | 双方确认资产已全部移交完毕 |
| **取消交接** | 任意时刻可取消，需二次确认 |

## 7. AI 助手（Assistant）

AI 助手基于课题组文档和历史对话，支持针对特定**课题**或**文档**进行问答。

### 7.1 开始对话

1. 在左侧**范围**选择器中选择课题或文档（决定 AI 的上下文范围）；
2. 在右侧输入框输入问题，点击**发送**；
3. AI 正在思考时显示动画占位气泡，回答完成后自动刷新显示结果；
4. 可在同一会话内继续追问（AI 会携带本次对话历史）。

**推荐使用场景：**

- "请帮我总结这份文档的核心内容"
- "这个课题目前有哪些待处理文档？"
- "这份报告有哪些需要改进的地方？"

### 7.2 会话管理

- 左侧会话列表显示历史会话，点击可切换；
- 每条会话右上角有**归档**按钮，可将不再活跃的会话归档；
- 点击**查看归档**可切换显示已归档会话，并可恢复。

### 7.3 记忆来源

AI 在回答时会自动携带：

- 本次会话的近期消息；
- 同范围内的历史回答；
- 你已**确认**的 AI 建议。

消息底部的折叠面板可查看记忆来源说明（如"2 条历史对话、1 条已确认建议"）。

## 8. 搜索与快速导航

页面顶部有**搜索 / 跳转**按钮（快捷键 `Ctrl+K`），可快速：

- 跳转到任意功能模块；
- 直接搜索文档标题并跳转到详情页。

## 9. 管理模块（仅管理员）

侧边栏底部的**管理**入口仅 admin 角色可见，包含四个标签页：

### 团队空间

创建和管理团队空间（顶层组织单元，如"2024级课题组"）。

### 项目管理

在团队空间下创建课题（项目），需指定课题名称、编号和负责人。

### 用户管理

- 创建新用户账号（设置用户名、密码、显示姓名、角色）；
- 修改用户信息和状态（启用 / 停用）。

**角色说明：**

| 角色 | 权限说明 |
|------|----------|
| `member` | 普通成员，可管理自己负责的文档 |
| `project_lead` | 课题负责人，额外可上传数据资产 |
| `admin` | 管理员，可使用管理模块 |

### 成员分配

将用户分配到具体课题，并指定其在课题内的角色（member / project_lead）。**只有分配到课题的成员才能看到该课题及其文档。**

## 10. 常见操作速查

| 场景 | 操作路径 |
|------|----------|
| 查看今日需处理的文档 | 总览 → 处理中 / 风险文档 |
| 上传新版文档 | 文档 → 点击文档名 → 上传新版本 |
| 转交文档给同事 | 文档详情 → 流转（处理中状态）→ 转交 → 选择成员 |
| 对文档提问 | AI 助手 → 选择文档范围 → 输入问题 |
| 生成文档摘要 | 文档详情 → 生成摘要 |
| 发起毕业交接 | 交接 → 创建交接 → 填写移交人和接收人 |
| 创建新用户 | 管理（admin）→ 用户管理 → 新建用户 |
