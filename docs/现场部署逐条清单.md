# DigiDocs Mgt 现场部署逐条清单

这份清单给到现场同事逐条执行。默认部署形态如下：

- DGX Spark：运行本系统、反向代理和 OpenClaw
- 群晖：运行 PostgreSQL，提供 DSM / File Station 文件存储底座
- 用户入口：只访问 DGX Spark，不直接访问群晖

## 1. 现场准备

- [ ] 确认 DGX Spark 的内网 IP
- [ ] 确认群晖的内网 IP
- [ ] 确认群晖 DSM HTTPS 端口为 `5001`
- [ ] 确认 DGX Spark 可以访问群晖的 `5432` 和 `5001`
- [ ] 确认普通用户只访问 DGX Spark 的 `80/443`
- [ ] 确认群晖的 `5432/5001` 只允许 DGX Spark 访问

## 2. 在群晖上准备数据库

- [ ] 在群晖 `Container Manager` 中创建 PostgreSQL 容器
- [ ] 使用 `postgres:17`
- [ ] 设置数据库名为 `digidocs_mgt`
- [ ] 设置独立数据库用户和强密码
- [ ] 把数据卷挂载到群晖持久化目录
- [ ] 将容器 `5432` 映射到群晖 `5432`
- [ ] 记录数据库连接串，供 DGX Spark 使用

示例连接串：

```text
postgres://digidocs:<强密码>@192.168.1.10:5432/digidocs_mgt?sslmode=disable
```

## 3. 在群晖上准备文件存储

- [ ] 创建共享目录 `DigiDocs`
- [ ] 创建服务账号 `digidocs_service`
- [ ] 给服务账号开通读、写、创建目录、上传、下载、删除权限
- [ ] 如需分享链接，再补充共享链接权限
- [ ] 确认 DSM / File Station API 可用
- [ ] 优先使用 HTTPS `5001`
- [ ] 群晖防火墙只放行 DGX Spark 的访问

说明：

- 这里使用的是群晖 DSM / File Station 作为文件存储底座
- 不是严格的 S3 对象存储

## 4. 在 DGX Spark 上准备运行环境

- [ ] 安装 Docker
- [ ] 安装 Docker Compose
- [ ] 安装 Git
- [ ] 安装 OpenClaw
- [ ] 确认 DGX Spark 能访问群晖数据库和 DSM

建议先检查连通性：

```bash
nc -vz 192.168.1.10 5432
nc -vz 192.168.1.10 5001
```

## 5. 启动 OpenClaw

- [ ] 按现场的 OpenClaw 安装方式完成安装
- [ ] 启动 OpenClaw Gateway
- [ ] 确认 `openclaw status` 正常
- [ ] 确认 OpenAI 兼容端点可用

必须验证：

```bash
curl -fsS http://127.0.0.1:18789/v1/models
```

说明：

- 只看 `openclaw status` 不够
- 本系统 Worker 还需要 `POST /v1/chat/completions`

## 6. 部署本系统到 DGX Spark

- [ ] 拉取仓库代码
- [ ] 复制 `.env.production.example` 为 `.env`
- [ ] 按现场实际环境修改 `.env`
- [ ] 将 `DATABASE_URL` 指向群晖 PostgreSQL
- [ ] 将 `STORAGE_BACKEND` 设为 `synology`
- [ ] 填好 `SYNOLOGY_HOST`、`SYNOLOGY_ACCOUNT`、`SYNOLOGY_PASSWORD`
- [ ] 确认 `CALLBACK_TOKEN` 与 `WORKER_CALLBACK_TOKEN` 一致
- [ ] 将 `OPENCLAW_BASE_URL` 指向 OpenClaw
- [ ] 设定 `JWT_SECRET` 为强随机值

启动命令：

```bash
docker compose --profile app up -d --build --remove-orphans
```

## 7. 初始化数据库

- [ ] 等待 `backend-go` 自动执行 migrations
- [ ] 确认数据库表结构已创建
- [ ] 如需演示数据，导入 `backend-go/sql/seed.sql`
- [ ] 初始化后立即修改默认演示账号密码

示例：

```bash
psql "postgres://digidocs:<强密码>@192.168.1.10:5432/digidocs_mgt?sslmode=disable" \
  -f backend-go/sql/seed.sql
```

## 8. 配置访问入口

- [ ] 先使用 DGX Spark 的 `18080` 做现场验收
- [ ] 如需正式对外访问，再在 DGX Spark 上加反向代理和 HTTPS
- [ ] 让用户只访问 DGX Spark，不直接访问群晖
- [ ] 不长期开放 `18080/18081` 给普通用户

临时访问示例：

```text
http://<DGX Spark 内网 IP>:18080
```

## 9. 现场验证

- [ ] `docker compose ps` 正常
- [ ] `curl -fsS http://127.0.0.1:18081/healthz` 正常
- [ ] `curl -fsS http://127.0.0.1:18080/` 正常
- [ ] `curl -fsS http://127.0.0.1:18789/v1/models` 正常
- [ ] 账号可正常登录
- [ ] 可以创建团队空间、项目和文档
- [ ] 可以上传文档
- [ ] 可以上传新版本
- [ ] 可以触发 AI 摘要或助手问答
- [ ] 可以访问群晖上的文件
- [ ] 可以读写群晖上的 PostgreSQL

建议再跑一次：

```bash
RUN_SYNOLOGY_PREFLIGHT=1 make smoke
```

- [ ] 记录 OpenClaw 的端点和模型名

- [ ] 留下升级命令给后续运维人员

